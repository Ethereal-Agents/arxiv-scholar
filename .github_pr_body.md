## Description
This PR introduces native server-side weighted fusion to our Qdrant hybrid search implementation, maximizing our retrieval performance for BGE-M3 and BM25.

### Changes Included
- **Configuration**: Added `DENSE_WEIGHT` (default: 0.6) and `SPARSE_WEIGHT` (default: 0.4) to `configs/config.py`.
- **Retrieval Engine**: Updated `HybridRetriever.retrieve()` to use Qdrant's native `models.RrfQuery` with custom weights, shifting from unweighted RRF to a highly tuned, low-latency fusion.
- **Diagnostics**: Expanded `run_benchmarks.py` with a powerful `run_alpha_sweep` diagnostic. This script fetches raw payloads, applies Min-Max normalization, iterates through alpha blends (0.0 to 1.0), and even cross-encodes candidate pools.
- **Documentation**: Added `alpha_sweep_results.md` outlining the empirical justification for the 0.6 / 0.4 split, proving that this ratio yields our maximum Recall@20 (0.8293) without the latency penalty of client-side fusion.
